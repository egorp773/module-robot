# Автономный робот-снегоуборщик - MVP

Модульный робот на гусеницах с GPS-навигацией и автономным режимом уборки территории.

## Архитектура

### Железо
- **ESP32** - основной контроллер
- **u-blox ZED-F9P** (2 шт) - RTK GPS модули (rover на роботе, base станция)
- **BNO085 IMU** - гироскоп/акселерометр для точного курса
- **Моторы гироскутера** - управление по UART
- **MAX98357A I2S** - звуковой модуль
- **Реле** - управление насадкой и креплением

### Софт
- **ESP32 прошивка** (`firmware/`) - модульная архитектура с GPS, IMU, PID-навигацией
- **Flutter приложение** (`module_app/`) - управление, запись территории, автономный режим

## Распиновка ESP32

| Модуль | Интерфейс | ESP32 пин | Примечание |
|--------|-----------|-----------|------------|
| F9P GPS (rover) | UART1 | GPIO4 (RX), GPIO5 (TX) | UBX-NAV-PVT + RTCM |
| Моторы | UART2 | GPIO16 (RX), GPIO17 (TX) | Протокол гироскутера |
| BNO085 IMU | I2C | GPIO21 (SDA), GPIO22 (SCL) | Rotation vector 50 Hz |
| MAX98357A | I2S | GPIO25/26/27 | Звук |
| Реле ATTACHMENT | GPIO | GPIO32 | Active HIGH |
| Реле MOUNT | GPIO | GPIO33 | Active HIGH |

## Установка и сборка

### 1. Прошивка ESP32

**Требования:**
- Arduino IDE 2.x
- ESP32 board support (Espressif Systems)
- Библиотеки:
  - `AsyncTCP`
  - `ESPAsyncWebServer`
  - `Adafruit_BNO08x`
  - `LittleFS`

**Шаги:**
1. Скопируйте `sound/data/` в `firmware/data/`
2. Загрузите файловую систему: Tools → ESP32 Sketch Data Upload
3. Откройте `firmware/firmware.ino` в Arduino IDE
4. Выберите плату: ESP32 Dev Module
5. Настройки:
   - Upload Speed: 921600
   - Flash Size: 4MB (3MB APP / 1MB SPIFFS)
   - Partition Scheme: Default 4MB with spiffs
6. Загрузите прошивку

**Проверка:**
```bash
# Подключитесь к Serial Monitor (115200 baud)
# Должны увидеть:
=== Robot Firmware v2.0 ===
GPS: UART1 38400 baud, RX=4 TX=5
IMU: BNO085 found
WiFi AP: 192.168.4.1
=== READY ===
```

### 2. Flutter приложение

**Требования:**
- Flutter SDK 3.3+
- Dart SDK 3.0+

**Шаги:**
```bash
cd module_app
flutter pub get
flutter run
```

**Для iOS:**
```bash
cd ios
pod install
cd ..
flutter run
```

**Для Android:**
```bash
flutter build apk --release
# APK: build/app/outputs/flutter-apk/app-release.apk
```

## Использование

### Базовая настройка

1. **Подключение к роботу:**
   - Включите робота
   - Подключитесь к WiFi сети "Robot" (пароль задаётся локально в прошивке)
   - Откройте приложение
   - Нажмите "Подключить"

2. **Проверка GPS:**
   - На главном экране должен появиться GPS индикатор
   - Тип фикса: 3D (без RTK) или RTK Fixed (с базовой станцией)
   - Точность: <50мм для RTK, <5м для 3D

### Запись территории (GPS-периметр)

1. Перейдите в "Ручное Управление"
2. Нажмите кнопку "Записать периметр" (появится красный индикатор)
3. Объезжайте границы территории вручную
4. Робот записывает GPS-точки каждые ~0.5м
5. Нажмите "Стоп запись"
6. Полигон упрощается (Douglas-Peucker) и сохраняется как зона
7. Сохраните карту с названием

### Автономная уборка

1. Перейдите в "Автоматический Режим"
2. Загрузите сохраненную GPS-карту
3. Нажмите "Построить маршрут" (змейка с шагом 50см)
4. Нажмите "Отправить маршрут" → робот загружает точки
5. Нажмите "Старт" → робот начинает движение
6. Управление: Пауза / Продолжить / Стоп
7. Прогресс: точка N из M, расстояние до следующей точки

### RTK-коррекция (опционально)

Для точности ~2см нужна базовая станция:

1. Подключите второй F9P к компьютеру/RPi
2. Настройте его как базу (Survey-In mode)
3. Запустите RTCM-сервер (например, `str2str` из RTKLIB):
   ```bash
   str2str -in serial://ttyUSB0:38400 -out tcpsvr://:2101
   ```
4. Робот автоматически получает RTCM по UDP порт 2101

## Протокол WebSocket

### Приложение → ESP32

| Команда | Формат | Описание |
|---------|--------|----------|
| Движение | `M,<left>,<right>` | Ручное управление (-100..100) |
| Стоп | `STOP` | Остановка |
| Начало маршрута | `ROUTE_BEGIN,<N>` | Подготовить N точек |
| Точка | `ROUTE_WP,<i>,<lat>,<lon>` | Добавить точку (8 знаков) |
| Конец маршрута | `ROUTE_END` | Завершить загрузку |
| Старт | `NAV_START` | Начать автономное движение |
| Пауза | `NAV_PAUSE` | Пауза |
| Продолжить | `NAV_RESUME` | Продолжить |
| Стоп авто | `NAV_STOP` | Остановить автономный режим |

### ESP32 → Приложение

| Сообщение | Формат | Частота |
|-----------|--------|---------|
| GPS | `GPS,<lat>,<lon>,<heading>,<fixType>,<hAcc>` | 5 Гц |
| IMU | `IMU,<yaw>` | 5 Гц |
| Батарея | `BAT_PCT,<int>` | 2 Гц |
| Навигация | `NAV,<state>,<wpIdx>,<wpTotal>,<distToWp>` | 2 Гц |
| Точка достигнута | `NAV_WP,<index>` | По событию |

## Настройка параметров

### PID-контроллер (config.h)

```cpp
#define PID_KP_HEADING    2.0f   // Пропорциональный коэффициент
#define PID_KI_HEADING    0.1f   // Интегральный коэффициент
#define PID_KD_HEADING    0.5f   // Дифференциальный коэффициент
```

Тюнинг на поле:
- Увеличьте Kp если робот медленно поворачивает
- Уменьшите Kp если робот колеблется
- Увеличьте Kd для демпфирования колебаний
- Ki обычно минимален (анти-windup)

### Навигация (config.h)

```cpp
#define ARRIVAL_RADIUS    0.15f   // Радиус достижения точки (м)
#define NOMINAL_SPEED     30      // Базовая скорость (-100..100)
#define MIN_GPS_ACC_MM    500     // Макс. допустимая погрешность GPS (мм)
```

### Маршрут змейки (Flutter)

В `cleaning_route_planner.dart` передайте `lineStep = 0.50` (метры) для GPS-карт.

## Troubleshooting

### GPS не работает
- Проверьте подключение F9P: RX=4, TX=5
- Откройте Serial Monitor → должны быть сообщения "GPS: UBX-NAV-PVT"
- Вынесите робота на открытое пространство (видимость неба)
- Подождите 1-2 минуты для холодного старта

### IMU не работает
- Проверьте I2C: SDA=21, SCL=22
- Serial Monitor → "IMU: BNO085 found"
- Если "not found" → проверьте питание 3.3V и подтяжки I2C

### Робот не следует по маршруту
- Проверьте GPS accuracy < 500мм
- Проверьте IMU yaw (должен меняться при вращении)
- Увеличьте NOMINAL_SPEED если робот слишком медленный
- Тюнингуйте PID параметры

### WebSocket отключается
- Проверьте WiFi сигнал
- Убедитесь что телефон подключен к сети "Robot"
- Перезагрузите ESP32
